UUID Version Reference

  • v1 (Timestamp-based): Embeds the current timestamp into the UUID structure. Simulated here in the browser — time bits are derived from Date.now() with random clock sequence and node ID.
  • v4 (Random): Entirely randomly generated using crypto.getRandomValues(). Practically zero collision probability. The recommended default.
  • v5 (Name-based SHA-1): Deterministic — the same namespace + name always produces the same UUID. Uses SHA-1 hashing via CryptoJS. Useful for reproducible IDs.
  • Prefix / Suffix: Prepend or append custom strings (e.g. user- / -prod) to every UUID in the output.
  • SQL Snippet: Wraps all UUIDs into a ready-to-run INSERT INTO … VALUES … statement.
